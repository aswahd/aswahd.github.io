Learning Affinity

  • AffinityNet learns the semantic affinity between a pair of adjacent coordinates

Affinity:
| Whether two pixels belong to the same instance or not.

Semantic Affinity:
| Whether two pixels belong to the same class or not.

  • AffinityNet predicts a convolutional feature map faff

  • the semantic affinity between a pair of feature vectors is then defined as

    Wij = exp{ − ∥faff(xi, yi) − faff(xj, yj)∥1},

Q. How to train AffinityNet?
What we have: image-level class labels
What we want: pairwise per-pixels semantic affinity labels

Generating Semantic Affinity Labels

Assumption: training semantic affinities from non-exhaustive pixels can generalize to other pixels

Idea: Identify confident areas of objects and background from CAMs

Identifying Confident Areas of Objects:

  • Amplify background activation maps so that background scores dominate insignificant activation scores of objects by decreasing $\alpha$ in $$M_{bg}(x, y) = \{1 - \underset{c \in C}{\max}M_c(x, y)\}^\alpha, \alpha = 4$$

  • Apply dCRF to refine the CAMs (to better estimate object shapes)

  • Collect the coordinates whose scores for the target class are greater than those of any other class including the amplified background

Identifying Confident Background Areas:

  • Weaken background activation maps by increasing $\alpha$ in $$ M_{bg}(x, y) = \{1 - \underset{c \in C}{\max}M_c(x, y)\}^\alpha $$

  • Confident background areas are regions with the least activation score among all classes

  • Remaining areas are considered neutral (ignored during training)

Generating Semantic Affinity Labels

AffinityNet Training

  • Affinities of only sufficiently adjacent coordinates are considered during training

    Q. Why?

    • Lack of context if two coordinates are too far from each other

    • To reduce the computational cost

Denote the pairs of pixels used during training as 𝒫, where 𝒫 = {(i, j)|d((xi, yi), (xj, yj)) < γ, ∀i ≠ j}, where d is the Euclidean distance, and γ is a radius that limits the distance between pairs of pixels.

Class Imbalance:

  • Most affinities are positive

  • Furthermore, the number of background pairs in the positive affinities is significantly larger

AffinityNet Training

  • Separate 𝒫 into two subsets; positive and negative pairs.

    $$\begin{aligned} \mathcal{P}^+ &= \{(i,j) | (i, j) \in \mathcal{P}, W^*_{ij} = 1\} \\ \mathcal{P}^- &= \{(i,j) | (i, j) \in \mathcal{P}, W^*_{ij} = 0\}, \end{aligned}$$ where

    $$W^*_{ij} = \begin{cases} 1 & \text{if $(x_i, y_i)$, and $(x_j, y_j)$ belong to the same class, and are not neutral} \\ 0 & \text{otherwise} \\ \end{cases}$$

  • Further divide the positive affinities into foreground and background,

$$\begin{aligned} \mathcal{L}_{fg}^+ &= -\frac{1}{\|\mathcal{P}_{fg}^+\|} \sum\limits_{(i,j) \in \mathcal{P}_{fg}^+} \log W_{ij} \\ \mathcal{L}_{bg}^+ &= -\frac{1}{\|\mathcal{P}_{bg}^+\|} \sum\limits_{(i,j) \in \mathcal{P}_{bg}^+} \log W_{ij} \\ \mathcal{L}^- &= -\frac{1}{\|\mathcal{P}^-\|} \sum\limits_{(i,j) \in \mathcal{P}^-} \log (1 - W_{ij}) \\ \end{aligned}$$

The affinity loss is defined as, ℒ = ℒfg+ + ℒbg+ + 2ℒ−

Random Walk Graphs

Motivation:
Given a weighted graph and a population, we would like to know the percentage of the population at each node after having the walkers roam the graph for a very long time.

i.e., for a node X, we want to find Pr(x).

Q. Calculate P(A), P(B), and P(C)

Analytic Solution $$\begin{aligned} P(A) &= P(B) + 0.5 * P(C) \\ P(B) &= 0.5 * P(A) + 0.5 * P(C) \\ P(C) &= 0.5 * P(A) \\ P(A) &= 0.44 \\ P(B) &= 0.33 \\ P(C) &= 0.22 \\ \end{aligned}$$

The analytical solution is computationally expensive for large graphs (for example, 512 x 512 graph).

Efficient solution

  1. Start with a population of 100, and divide them equally across the nodes.

  2. Take one random walk, moving a proportion of the population from each node to every one of its neighboring nodes in accordance with the weight on the edges.

  3. Go to 2 until convergence or a threshold

  • Of the 100 walkers, 44.44 of them will end up in A, 33.33 in B, and 22.22 in C.

  • Random walk graphs can be used to rank the popularity of web pages.

How to Use Random Walks in Semantic Segmentation:

  • Propagate the activations to nearby and semantically identical areas

  • And penalize the propagation to areas of other classes

Random Walk Graphs

  • Let G(V, E) denote an undirected graph with a set of vertices V and a set of edges E.

  • Then a random walk can be characterized by the transition probabilities between its vertices.

Let W be a symmetric nxn affinity matrix, where n denotes the number of nodes, and Wij ∈ [0, 1] denotes how similar the nodes i and j are.

In the case of semantic segmentation, each pixel can be regarded as a node.

Random Graph Walk Let D denote an nxn diagonal matrix, where $D_{ii} = \sum_{j=1}^n W_{ij}, \forall i \ne j$.

Then a random walk transition probability can be described as A = D − 1W. i.e.,

The weights for a particular node in the affinity matrix don’t sum up to 1. Therefore, make the probabilities using A = D − 1W.

Example: Page ranking
Let yt be a nx1 vector denoting a node distribution at time t.

According to random walk theory, we can spread the information by performing a one-step random walk described by yt + 1 = Ayt

  • Repeat the operation until convergence.

Similarly,

  • semantic affinities can be used to spread the CAMs using random walk.

Q. How?

  1. Predict convolutional feature maps using AffinityNet

  2. Compute the affinities from the features within local circle of radius γ

  3. Compute the transition probability from the affinities as follows

T = D − 1W ∘ β,  where Dii = ∑jWijβ, where

  • W ∘ β - Hadamard power of β (β > 1, to eliminate weak affinities)

shapes: D ∈ (n2xn2), W ∈ (n2xn2), T ∈ (n2xn2)

  • Apply random walk

    vec(Mc*) = Tt. vec(Mc), ∀c ∈ C ∪ bg,

where

  1. vec - vectorization of a matrix

  2. t - the number of iterations

  • Apply random walk until a predefined number of iterations.

Learning a Semantic Segmentation Network

The refined CAMs are used to generate pseudo ground truth masks.

  • first upsample the refined CAMs to the resolution of input image using bilinear interpolation

  • for each pixel, select the class label with the maximum activation

  • finally, the (pseudo) segmentation labels are used to train a fully supervised segmentation model

Network Architectures

Backbone Network:

  • ResNet38 Backbone

    • Remove the GAP and fully connected layers

    • Replace the last three convolutions by atrous convolutions

    • The subsampling ratio is 1/8

CAM Network:

  • Add a 3x3 convolution, a GAP, and fully connected layer to the backbone network

Affinity Network:

  • Aggregate features from the last three levels of the backbone network

  • Add a 1x1 convolution on top of the aggregated features

Segmentation Model:

  • Add two more atrous convolution layers (with r = 12) to the backbone network
